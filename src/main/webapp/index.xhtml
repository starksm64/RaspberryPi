The Raspberry Pi ® is a low cost(~$35), credit-card sized computer that plugs into a computer monitor or TV, and uses a standard keyboard and mouse. It is a capable little device that enables people of all ages to explore computing, and to learn how to program. It’s capable of doing everything you’d expect a desktop computer to do, from browsing the internet and playing high-definition video, to making spreadsheets, word-processing, and playing games. Your reading this from a Raspberry Pi running Java® SE 7 and the JBoss® Java® EE7 Wildfly implementation.

You may access your Raspberry Pi by sshing into it as follows: ssh -l jbosspi #{piApp.ipAddress} using jbosspi as the password when you are prompted.

The jbosspi user has sudo access so you can run priviledged commands. You may also simply su to root or login as root. The root password is also jbosspi.

The operating system is a version of Fedora 18 called pidora. Once you are logged in, you can explore the installation, check on the Wildfly server status, use yum to install additional packages, etc. The Wildfly installation is located in the ~jbosspi home directory. You can control the server using the service command to run the /etc/init.d/wildfly script:

[jbosspi@raspi ~]$ service wildfly Usage: /etc/init.d/wildfly {start|stop|status|restart} [jbosspi@raspi ~]$ service wildfly status wildfly is running (pid 4450)

The github project for the Hacknight application on the Raspberry Pi has been cloned into the ~jbosspi home directory. Building on the Pi with maven is rather slow, so what I typically do is build on my laptap, and then scp the war over:

scp target/iotweb.war root@mypi:/home/jbosspi/wildfly-8.0.0.Final/standalone/deployments
